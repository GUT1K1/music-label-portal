# Инструкция по сборке проекта

## Автоматическая генерация sitemap.xml

Перед каждым билдом проекта необходимо обновить `sitemap.xml` с актуальными статьями блога.

### Способ 1: Автоматический (рекомендуется)

```bash
# Запустить скрипт генерации sitemap
node scripts/generate-sitemap.js

# Затем собрать проект
npm run build
```

### Способ 2: Через веб-интерфейс

1. Откройте страницу `/generate-sitemap` в браузере
2. Дождитесь загрузки статей
3. Скачанный `sitemap.xml` замените в папке `public/`
4. Соберите проект

### Способ 3: Bash скрипт (для CI/CD)

```bash
# Сделать скрипт исполняемым
chmod +x scripts/prebuild.sh

# Запустить перед билдом
./scripts/prebuild.sh && npm run build
```

## Что обновляется в sitemap

- ✅ Текущая дата для всех основных страниц
- ✅ Все статьи блога из API
- ✅ Актуальные даты публикации статей
- ✅ Правильные приоритеты и частота обновлений

## Проверка sitemap

После генерации проверьте `public/sitemap.xml`:
- Должно быть 7 основных страниц + все статьи блога
- Даты должны быть актуальными
- Все URL должны начинаться с `https://420music.ru/`

## Для разработчиков

Скрипт генерации находится в `scripts/generate-sitemap.js` и:
- Загружает статьи из API
- Формирует валидный XML
- Сохраняет в `public/sitemap.xml`
- Выводит статистику в консоль
